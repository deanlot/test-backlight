<p>Note how "disabled" state always takes priority over "busy" state.</p>
<Playground
  scope={{ RadButton }}

  `}
/>

### Notify Success onClick (Promise)
<p>
  "onClick" can receive either a promise or a synchronous callback. Below is an example of a promise-based callback.
  This will trigger a success icon to be displayed for 1200ms on resolution of the promise
</p>
<Playground
  scope={{ RadButton }}
  code={`
  <RadButton
    variant="primary"
    notifySuccess
    onClick={() => new Promise((resolve) => resolve())}
  >
    Click Me
  </RadButton>
  `}
/>

### Notify Success onClick (Synchronous)
<p>
  Just like the promise-based version above, "onClick" can receive a synchronous callback that will be turned into a promise
  inside of the button. I think
</p>
<Playground
  scope={{ RadButton }}
  code={`
  <RadButton
    variant="primary"
    notifySuccess
    onClick={() => {}}
  >
    Click Me
  </RadButton>
  `}
/>


### Notify Failure onClick
<Playground
  scope={{ RadButton }}
  code={`
  <RadButton
    variant="primary"
    notifyFailure
    onClick={() => new Promise((resolve, reject) => reject())}
  >
    Click Me
  </RadButton>
  `}
/>
